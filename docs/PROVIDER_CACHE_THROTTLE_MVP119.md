# MVP119：Provider 缓存、节流与手动回退

## 行为

1. 单个 RJ 的 DLsite 成功结果在 Electron main 内存中缓存 10 分钟。
2. “查询（优先缓存）”先读取缓存，命中时不发送网络请求。
3. “重新查询”强制尝试网络访问，但同一 RJ 的联网请求至少间隔 5 秒。
4. 页面显示最近查询时间、结果来源、缓存写入时间和有效期。
5. “清除缓存”只删除短期内存缓存，不删除当前差异预览，也不删除本地元数据覆盖。
6. 网络、超时、HTTP 或解析失败时，用户仍可粘贴标准 JSON 生成差异预览。

## 安全边界

- 只查询单个 RJ。
- 不做全库批量抓取。
- 不自动保存 Provider 候选值。
- 不自动覆盖本地元数据。
- 不修改媒体文件或音频标签。
- 不接 SQLite。
- 不向 Renderer 返回 absolutePath 或 file://。
- 缓存只存在于当前 Electron 进程，应用退出后自然清空。
