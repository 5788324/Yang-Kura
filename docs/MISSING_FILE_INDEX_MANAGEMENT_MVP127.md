# MVP127 缺失文件与失效索引管理

## 目标

在不修改媒体文件的前提下，对当前 `library-index.json` 做只读检查，并给用户提供安全的后续处理入口。

## 已实现

- 只读检查音轨、封面、字幕的相对路径。
- 识别缺失文件、无法读取、路径指向目录、非法路径和集合失效引用。
- Renderer 只接收相对路径、记录 ID 和统计信息。
- 可以打开失效记录最近仍存在的目录，方便用户手动检查是否移动或改名。
- 可以生成“从索引移除预览”，列出计划移除的音轨记录、封面引用、字幕引用和失效集合引用。
- 可以调用既有 dry-run 入口执行只读重新扫描。

## 安全边界

- 检查过程不写 `library-index.json`。
- 移除功能目前只有预览，不执行真实写入。
- 不会删除真实文件。
- 不会移动、重命名或覆盖媒体文件。
- 不返回 `absolutePath` 或 `file://`。
- 不接 SQLite。
- 不自动扫描未由用户授权的目录。

## 后续

真实索引清理应单独增加：二次确认、旧 index 备份、SHA-256 一致性复核、写入后读回校验。该能力不属于 MVP127。
