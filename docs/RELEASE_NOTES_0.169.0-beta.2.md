# Yang-Kura 0.169.0 Beta 2 · 个人日用版

> 平台：Windows x64  
> 发布形式：GitHub prerelease  
> 升级来源：`0.168.0-beta.1`

这是 Yang-Kura 完成媒体库正式页面迁移后的个人日用版本。该版本继续面向个人本地使用，不商业化、不对外分发，不引入云端账号、多人权限或企业级治理。

## 下载资产

- `Yang Kura-0.169.0-beta.2-portable-x64.exe`：便携版。
- `Yang Kura-0.169.0-beta.2-setup-x64.exe`：NSIS 安装版。
- `SHA256SUMS.txt`：两个可执行文件的 SHA-256 校验值。

## 本次主要变化

### 正式媒体库界面

- 首页、音声库、RJ 详情、音乐库全部迁移到正式 Design System。
- 音乐库提供歌曲、专辑、艺术家、文件夹四种视图。
- 专辑、艺术家、文件夹支持钻取详情、返回导航、播放全部和全部加入队列。
- 主要列表统一使用共享 `MediaCard` / `TrackRow`。
- 支持搜索、排序、收藏、多选、全选当前结果和批量加入歌单或播放队列。
- 深色“暮夜琥珀”和浅色“雾光象牙”主题覆盖主要日常页面。
- 完成窄窗口、键盘焦点、空状态、错误恢复和 reduced-motion 适配。

### RJ 详情与元数据

- 正式 RJ 详情页保留播放、队列、收藏、字幕状态、外部打开和文件管理器定位。
- 本地元数据覆盖、清除覆盖、DLsite 单 RJ 查询、差异预览和字段选择性应用继续可用。
- 评分、收听状态和个人笔记仅写入本地覆盖层，不改写音频文件。

### 播放与媒体可靠性

- 继续支持 HTMLAudio、mpv 子进程后端与自动 fallback。
- 保留 Seek、队列、历史、续播、播放结束策略和字幕/歌词。
- 支持 LRC、SRT、VTT、ASS 与双语字幕。
- 修正音乐库批量加入队列的实时状态验收，防止批量操作丢项。
- 修正音乐曲目标题与艺术家信息的行布局。

### 导入、索引与数据安全

- Local JSON Index 写入、读取、备份、恢复和维护保持不变。
- copy-only 与受控 move-only 导入事务继续提供失败回滚和 OperationLog。
- 缺失文件检查、受控索引清理、备份保留和维护历史继续可用。
- Renderer 继续使用 rootPathToken 与相对路径，不暴露不必要的绝对路径或 `file://`。
- 不自动覆盖或删除真实媒体文件。

## 升级与数据建议

- 从 Beta 1 升级时可直接安装 Beta 2；安装器配置为保留用户数据。
- 覆盖安装前仍建议备份重要的 `library-index.json`、本地元数据覆盖和自定义歌单。
- 便携版与安装版不要同时运行，避免同一媒体库被两个进程同时操作。
- 第一次打开后，如本地目录授权失效，请重新选择资源库并读取现有 Index。

## 已知限制

- 未进行商业代码签名，Windows 可能显示未知发布者或 SmartScreen 提示。
- 不包含自动更新；后续版本需要手动下载安装。
- mpv、声卡、驱动和特殊音频格式组合无法在所有真实设备上穷举验证；不可用时会回退到 HTMLAudio。
- DLsite Provider 依赖网络和目标站点状态，失败时仍可使用手工元数据。
- 不提供正式下载器、SQLite 全面迁移、OpenList/WebDAV、云同步、在线账号或插件市场。
- 不包含完整 AI Agent、Arsm_Transcribe 正式集成或 Player Core v2。
- 该版本是个人日用 prerelease，不承诺商业发行级兼容性与支持周期。

## 安全边界

- 媒体文件不会因扫描、浏览或元数据编辑被自动改写。
- Index 写入保留备份和读回校验。
- 导入、清理、恢复和安装验收使用临时目录或副本。
- Provider 查询不会自动覆盖用户本地元数据。
