没有通过脚本、诊断入口、配置注入或手工修改 Index 绕过该 GUI 状态。

## 问题清单

### MAJ-001：资源库授权与 Index 状态断裂

- 严重度：Major。
- 模块：设置页、资源库授权、Local JSON Index、浏览页与播放器状态。
- 复现：设置 → 资源库目录 → 选择音声库 `E:\arsm` → 音声库。
- 实际：目录选择提示成功，但授权摘要、读取/扫描能力、浏览计数和播放器数据不一致。
- 预期：已授权目录应允许读取已有 Index，或在无 Index 时允许安全扫描；各页面资源计数应来自同一真实数据快照。
- 影响：无法继续真实播放、Seek、字幕、队列、歌单添加和重启恢复验收。

### MAJ-002：诊断资源状态仍为 Demo

- 严重度：Major。
- 模块：AI 维护 → 诊断工具 → 刷新资源状态。
- 实际：刷新后显示 Demo 扫描并声明不会读取真实磁盘，资源计数为 0。
- 预期：诊断页应反映已授权 root、真实 Index 与当前 App 资源状态；若真实诊断尚不可用，应禁用并明确说明，不得把 Demo 结果作为当前状态。

### MIN-001：导入器初始页示例容易被误解

- 严重度：Minor。
- 现象：未选择来源时仍显示“4 个示例文件”和“0 阻断 / 1 提醒”。
- 处理：不属于当前 Major 根因，等待资源库主链修复后，在真实导入流程中复核。

### OBS-001：原生目录选择器自动化限制

- 性质：测试工具限制。
- 处理：用户人工完成 Windows 原生目录选择，其余流程由 Codex 继续验证。

- `npm audit`：0 high / critical，1 moderate。
