# MVP126 大资源库性能评估

## 测试规模

- 4,000 个音声作品。
- 每个音声作品 8 条曲目，共 32,000 条。
- 1,500 个音乐专辑。
- 每个音乐专辑 12 条曲目，共 18,000 条。
- 合计 50,000 条曲目。

测试命令：

```bash
npm run test:library:performance
```

测试数据完全在内存中生成，不读取用户目录，也不写磁盘索引。

## UI 降载策略

| 页面 | 首批渲染 | 单次追加 |
|---|---:|---:|
| 音声作品 | 72 | 72 |
| 音乐曲目 | 160 | 160 |
| 音乐专辑 | 72 | 72 |
| 艺术家/文件夹 | 120 | 120 |

搜索结果总数不会截断，仅限制同时存在的 DOM 节点。

## 代码分割

- 主页面按路由 `React.lazy` 加载。
- 完整诊断模块只有在用户点击“打开完整诊断”后加载。
- React/ReactDOM 与 Lucide 图标使用独立 vendor chunks。

## 安全边界

- 不读取真实媒体库。
- 不产生 `absolutePath` 或 `file://`。
- 不修改媒体文件。
- 不修改 Local JSON Index。
