# U25 工程窗口与历史验收卡清理

## 目标

继续执行项目级日常 UI 硬规则：用户界面只展示真实可操作功能，历史开发说明、回归记录和命令行验收窗口不得继续占据首页与设置页。

## 清理范围

### 首页

- 隐藏历史“日常首页 / 日常使用”说明卡；
- 保留真实的继续播放、最近播放、资源库、音声库、音乐库和歌单入口；
- 历史 ID 和数据模型继续保留为 verifier 兼容层。

### 播放设置

- 隐藏 mpv 稳定性内部状态；
- 隐藏 Windows 播放链路命令行快检；
- 隐藏源码包小样本验证命令；
- 保留播放后端偏好、mpv 检测、选择与清除功能。

### 资源库设置

- 隐藏“设置页精简”“日常设置检查”“GitHub 基线”等历史说明卡；
- 隐藏 Scanner Contract 的 Allowed/Blocked、状态枚举和开发阶段说明；
- 保留目录选择、快速导入、扫描、索引读取/写入、备份和用户结果提示。

### 关于页

- 隐藏组件健康、回归修复、Electron 加固、Beta GUI、RC、发布候选和交接记录；
- 保留产品名称、定位、当前阶段和个人本地隐私说明。

## 实现策略

- 历史窗口继续保留在源码中，但统一增加 `hidden` 与 `aria-hidden="true"`；
- 不删除服务、ID、verifier 锚点或维护路由；
- 后续有排障需求时，由 AI 维护读取底层状态，而不是重新暴露到普通设置页。

## 验证要求

U25 必须同时验证工程窗口默认不可见、真实用户功能仍可见、AI 维护路由仍存在，并通过全部历史稳定回归与生产构建。

## 边界

本轮不修改媒体扫描、Local JSON Index、播放器、mpv 后端、字幕、队列、导入、下载器、诊断页、Electron、持久化数据、主题色板或用户文件。
