# U24 日常侧栏去工程化与 AI 维护入口

## 目标

日常使用者只需要媒体库、歌单、导入和设置，不应长期看到下载规划、诊断、回归或检修入口。U24 调整的是可见层级，不删除底层排障能力。

## 当前项目进度

```text
核心版本：0.167.0-mvp129
U02～U08：产品化增量已完成
U09～U23：结构与质量优化已完成
U24：日常侧栏去工程化已完成
当前阶段：Windows GUI 日常流程验收与定向缺陷修复
MVP130：继续冻结，禁止合入
```

## 改动

- 默认侧栏标题改为“日常使用”；
- “系统设置”进入默认可见的日常入口；
- “下载规划”和“诊断工具”移入默认折叠的“AI 维护”；
- AI 维护副标题明确为“工程与检修工具”；
- 折叠按钮提供 `aria-expanded` 与 `aria-controls`；
- 当前页面属于下载或诊断时，维护区自动保持可见；
- 页面类型、App 路由、懒加载组件和底层功能全部保留。

## 用户结果

正常启动时，侧栏不再把工程页与媒体入口放在同一视觉层级。用户只有主动展开“AI 维护”时才会看到检修工具；AI 或开发排障仍可使用原页面。

## 边界

本轮不修改资源扫描、Local JSON Index、播放、字幕、队列、导入、下载器实现、诊断实现、Electron、主题色板或持久化数据。
