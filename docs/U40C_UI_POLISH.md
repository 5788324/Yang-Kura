# U40-C UI 细节收口

## 目标

U40-C 处理 U40-B 截图复核中确认的四项非阻断视觉问题：

1. 浅色 AI 维护页局部状态色和次级操作辨识度偏弱；
2. 音乐库专辑、艺术家和文件夹只有少量结果时，卡片被拉伸且留白失衡；
3. 全屏歌词与主应用的圆角、材质和品牌视觉不够统一；
4. 窄窗口设置页首屏信息密度偏高。

本轮只调整现有视觉表层，不改变播放器、资源库、Index、导入器、文件写入、安装器或发布逻辑。

## 实现

- `src/styles/u40c-ui-polish.css`
  - AI 维护状态、标签、按钮和禁用态使用语义 Token；
  - 音乐集合网格使用紧凑自适应宽度；
  - 全屏歌词统一主应用边框、圆角、阴影和品牌材质；
  - 移除全屏歌词装饰窗口圆点；
  - `1120px` 以下压缩设置维护横幅并保持设置标签单行滚动；
  - 保留 `620px` 单列回退及减少动画处理。
- `src/main.tsx`
  - 在既有主题与页面样式之后加载 U40-C 覆盖层。
- `scripts/verify-u40c-ui-polish.mjs`
  - 检查入口导入、关键页面选择器、语义 Token、响应式和减少动画规则。
- `.github/workflows/u40c-ui-polish.yml`
  - 提供低风险 UI 修改的 Windows 定向验收。

## 排除项

按用户要求，本轮不进行物理声卡、扬声器、耳机、真实显示器主观观感或第三方程序界面测试。

## 发布判断

U40-C 不修改产品行为、依赖、Electron Main、安装器或用户数据格式，因此：

- 版本继续保持 `0.169.0-beta.2`；
- 既有 Beta 2 Release 不变；
- 不重新构建 portable 或 NSIS；
- 合并后恢复按需日常维护，不预排连续内部治理轮次。
