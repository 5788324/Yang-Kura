# Yang-Kura UI Design System

> 文档状态：规则级设计基线 1.0  
> 适用版本：`0.168.0-beta.1` 之后的 UI 重构与主题扩展  
> Beta 2 目标：`0.169.0-beta.2`  
> 核心原则：业务能力保留，界面分层重写；主题不是换色；动画不是装饰；任何新 UI 必须能够长期维护。

## 1. 文档地位

本文件是 Yang-Kura UI 设计和验收的最高优先级规则，用于约束：

- App Shell、导航、布局和信息架构；
- 首页、音声库、音乐库、详情、歌单、导入器、设置和 AI 维护；
- PlayerBar、经典、黑胶、歌词三种播放器；
- 深色、浅色和未来主题；
- 组件、排版、图标、动画、反馈和无障碍；
- 真实功能与 UI 入口的映射；
- Stitch、Lovable、AppDeploy、ChatGPT、Codex 和其他 AI 的生成结果；
- UI PR、截图审查和发布候选验收。

任何设计稿或 UI 代码与本文冲突时，以本文为准。修改本文必须说明原因、影响页面、迁移方式和回退方式。

## 2. 产品定位

Yang-Kura 是个人使用的 Windows 本地音频媒体库，重点支持：

1. ASMR / RJ 音声作品；
2. 普通本地音乐；
3. 统一 Track 播放器；
4. LRC / SRT / VTT / ASS 字幕；
5. 歌单、队列、历史和续播；
6. 安全导入、元数据维护和资源库检修。

它不是：

- SaaS Dashboard；
- 文件管理器换皮；
- 在线流媒体商城；
- 工程诊断控制台；
- 只有渐变、玻璃和假按钮的 AI Demo。

目标体验：安静、成熟、可信；内容优先；长时间使用不疲劳；大库仍然清晰；复杂功能不暴露工程细节。

## 3. 核心设计原则

### 3.1 内容先于容器

- 优先展示封面、作品标题、音轨、字幕、进度和状态；
- 不把所有信息包进同样大小的 Card；
- 列表优先使用留白、列对齐和分隔线；
- Card 只用于明确层级、点击范围或独立模块；
- 页面不能变成卡片矩阵 Dashboard。

### 3.2 日常层与工程层分离

日常层允许：

- 首页、音声库、音乐库、歌单、导入、设置；
- 搜索、队列、历史、收藏、字幕和播放器；
- 用户可理解的资源库状态和恢复操作。

工程层统一进入 AI 维护：

- Index 原始检查、备份、恢复和清理；
- 缺失文件和异常记录；
- mpv / HTMLAudio / fallback 诊断；
- Provider 缓存和错误；
- 导入事务和回滚日志；
- 版本、构建、MVP、verifier、Git 和 IPC；
- 原始错误码、调试 JSON 和绝对路径。

### 3.3 主题改变材质，不改变结构

主题必须改变：

- Canvas；
- Surface 层级；
- 字体色温；
- Border；
- Shadow / Elevation；
- 封面处理；
- 浮层和播放器环境；
- 反馈色在该环境中的表现。

主题不得改变：

- 页面信息架构；
- 控件位置和尺寸；
- 功能语义；
- 焦点顺序；
- 键盘行为；
- 功能可见性。

### 3.4 动画解释状态变化

允许：

- 页面层级切换；
- PlayerBar 展开；
- Drawer、Dialog、Search Overlay；
- Hover、Press、Focus；
- 当前歌词和当前播放项变化；
- 筛选、排序和布局切换；
- 加载、成功、失败和回滚反馈。

禁止：

- 每次打开全部元素依次飞入；
- 持续发光、呼吸、粒子和漂浮；
- 没有状态含义的弹跳和旋转；
- 不同组件使用不一致动画；
- 动画阻塞交互；
- reduced-motion 下仍持续旋转黑胶。

## 4. 信息架构

### 4.1 一级导航

固定顺序：

1. 首页；
2. 音声库；
3. 音乐库；
4. 歌单；
5. 导入资源；
6. 设置。

“下载规划、诊断、检修、回归、版本状态”不得回到一级导航。

### 4.2 快速入口

允许在首页或侧栏二级区域提供：

- 最近播放；
- 我的收藏；
- 播放历史；
- 未听完；
- 最近加入；
- 含字幕作品。

### 4.3 全局搜索

搜索范围：

- RJ 号；
- 作品标题；
- 社团；
- 声优；
- 标签；
- 音轨；
- 艺术家；
- 专辑；
- 歌单。

搜索 Overlay 必须：

- 支持键盘打开和关闭；
- 按类型分组；
- 有空状态和最近搜索；
- 不展示原始路径；
- 结果可直接播放或打开详情。

## 5. App Shell

固定层级：

```text
Title / Window Controls
├─ Sidebar
├─ Top Bar
├─ Page Content
├─ Overlay Layer
└─ Persistent PlayerBar
```

规则：

- PlayerBar 永远是独立层，不侵入页面布局；
- 页面滚动不移动侧栏和 PlayerBar；
- Top Bar 只保留当前页面动作、搜索和必要状态；
- 窗口控件、拖拽区域和交互控件不可重叠；
- 最小窗口下允许收窄侧栏，但不隐藏核心功能；
- 页面组件不直接管理 Shell 状态。

## 6. 布局与密度

### 6.1 基础网格

- 4px 为最小单位；
- 常用间距：4 / 8 / 12 / 16 / 24 / 32 / 48；
- 页面左右安全区：24～40px；
- 内容最大宽度根据页面类型决定，不统一强制居中；
- 播放器和歌词页允许沉浸式全宽；
- 资源库列表优先使用可伸缩列和虚拟滚动。

### 6.2 圆角

- 小控件：6～8px；
- 输入框、按钮：8～10px；
- Card：10～14px；
- Dialog / Drawer：14～18px；
- 不使用所有元素 20px 以上的大圆角；
- 封面圆角应小于普通 SaaS 卡片，保留媒体感。

### 6.3 信息密度

提供三档：

- 舒适：封面和留白优先；
- 标准：默认；
- 紧凑：大库管理。

密度变化只影响间距、行高和封面尺寸，不改变功能和信息层级。

## 7. 排版

### 7.1 字体

优先系统可用字体，不打包外部字体文件。建议：

```css
font-family: Inter, "Segoe UI", "Microsoft YaHei UI", "Noto Sans CJK SC", sans-serif;
```

日文内容允许使用系统日文字体 fallback。

### 7.2 字级

| 语义 | 建议 |
|---|---:|
| 页面标题 | 28～34 |
| 主要区块标题 | 20～24 |
| 卡片标题 | 15～17 |
| 正文 | 14～15 |
| 次要信息 | 12～13 |
| 标签 | 11～12 |
| 歌词当前行 | 28～42 |

规则：

- 不靠全大写制造层级；
- 技术信息使用普通 UI 字体，不使用等宽字体污染日常层；
- 数字时间、码率等可使用 tabular numerals；
- 标题截断必须提供 Tooltip 或可展开详情。

## 8. 图标

统一使用 Lucide 风格线性图标。

- 导航：20px；
- 工具栏：18px；
- 紧凑按钮：16px；
- 描边：约 1.75；
- 选中态可以改变背景或文字，不通过混用实心图标实现；
- 禁止 Emoji、Unicode、Lucide 和其他图标库混用；
- 纯图标按钮必须有可访问名称和 Tooltip。

## 9. 主题系统

主题使用语义 Token，不允许组件写死主题颜色。

### 9.1 Token 分组

```text
color.canvas
color.surface.1
color.surface.2
color.surface.3
color.overlay
color.text.primary
color.text.secondary
color.text.muted
color.border.subtle
color.border.strong
color.accent
color.accent.hover
color.accent.soft
color.success
color.warning
color.danger
color.focus
shadow.low
shadow.medium
shadow.high
material.blur
material.grain
```

### 9.2 暮夜琥珀

定位：夜间聆听室、深色媒体库。

- 深炭灰 Canvas，不使用纯黑；
- 石墨和烟灰 Surface；
- 暖白文字；
- 低饱和琥珀为唯一全局强调色；
- 封面颜色只在播放器沉浸背景中使用；
- 边框低对比；
- 阴影短而深；
- 不使用蓝紫霓虹和全局渐变。

### 9.3 雾光象牙

定位：白天整理、长时间浏览、元数据维护。

- 温暖象牙 Canvas，不使用刺眼纯白；
- 奶油白一级 Surface；
- 暖灰米色二级 Surface；
- 深棕黑主文字；
- 暖灰褐次文字；
- 低饱和陶土棕强调色；
- 浅暖灰实体边框；
- 柔和自然阴影；
- 封面自然色彩，轻微提亮；
- 歌词页表现为纸张阅读环境。

### 9.4 新主题准入

新主题必须提交：

- 完整 Token；
- Canvas / Surface / Overlay 材质说明；
- 字体色温；
- 边框和阴影；
- 封面处理；
- 经典、黑胶、歌词截图；
- 首页、资源库、详情、导入和设置截图；
- 对比度和 focus-visible 检查。

只换 accent 的方案不得称为新主题。

## 10. 基础组件

### 10.1 Button

类型：Primary、Secondary、Ghost、Danger、Icon。

- 同一页面只允许一个主要 Primary 动作；
- 危险操作必须明确文本，不只用红色图标；
- Loading 期间保持宽度；
- 禁用态仍需可辨认；
- 点击区域不小于 32px，常规建议 36～40px。

### 10.2 Media Card

必须显示：

- 封面或明确 fallback；
- 标题；
- 作品类型；
- 关键次信息；
- 播放或完成状态；
- Hover 操作。

不得把所有元数据塞入卡片。详细字段进入详情页或列表模式。

### 10.3 Track Row

建议列：序号、标题、字幕、格式、时长、进度、操作。

- 当前播放项有明确但克制的状态；
- Hover 出现快捷操作；
- 支持多选；
- 列宽可伸缩；
- 紧凑模式不牺牲点击区域。

### 10.4 Dialog / Drawer

- Dialog 用于需要确认或集中注意的短流程；
- Drawer 用于队列、筛选、详情辅助面板；
- 打开后焦点进入；
- Tab 圈定；
- Escape 关闭；
- 关闭后返回触发控件；
- 移动和删除等高风险操作不允许点击遮罩直接关闭。

### 10.5 Empty / Error / Recovery

每个页面必须区分：

- 首次空状态；
- 搜索/筛选无结果；
- 数据正在加载；
- 权限失效；
- Index 损坏；
- 文件缺失；
- 操作失败；
- 可恢复和不可恢复错误。

空状态必须提供下一步，不显示工程术语。

### 10.6 Toast 与反馈

- 成功：短暂；
- 警告：可关闭；
- 失败：保留并提供恢复动作；
- 回滚和导入结果使用页面内摘要，不只弹 Toast；
- mpv fallback 属于非阻断提示，不应显示为致命错误。

## 11. 页面规则

### 11.1 首页

优先顺序：

1. 继续播放；
2. 未听完；
3. 最近加入；
4. 最近播放；
5. 音声库 / 音乐库快捷入口；
6. 用户可理解的资源库状态。

必须设计：

- 首次启动；
- 目录未授权；
- 空 Index；
- Index 损坏恢复；
- 新增文件待扫描；
- fallback 非阻断状态。

禁止展示版本、MVP、扫描原始日志和测试结果。

### 11.2 音声库

必须支持：

- 网格 / 列表；
- 搜索；
- RJ、社团、声优、标签；
- 播放状态、字幕状态、文件状态；
- 最近加入、最近播放、评分、总时长等排序；
- 多选和批量操作；
- 密度切换；
- 保存筛选器可后置。

批量操作：收藏、标签、加入歌单、标记状态、重新匹配元数据。删除真实文件不得作为普通批量操作。

### 11.3 RJ 详情

必须包含：

- 封面、RJ、标题、社团、声优、标签和说明；
- 继续播放；
- 收藏、评分、备注；
- 音轨列表；
- 字幕状态；
- 附属图片、文本和视频；
- 打开文件夹；
- 本地元数据编辑；
- DLsite 匹配和字段对比；
- 元数据备份、历史和恢复；
- 文件健康状态。

二级流程必须设计字段级选择性应用，禁止 Provider 一键覆盖全部本地信息。

### 11.4 音乐库

必须包含：

- 专辑、艺术家、音轨；
- 专辑详情；
- 艺术家详情；
- 播放、收藏、加入歌单；
- 标签和封面状态；
- 格式、码率、采样率可在列表或详情查看；
- 多选和排序。

### 11.5 歌单、队列和历史

必须包含：

- 新建、改名、说明、封面和删除歌单；
- 队列拖拽、移除、下一首播放和保存歌单；
- ASMR / 音乐类型标识；
- 播放历史单项和按时间清理；
- 清理前确认；
- 重启恢复说明。

### 11.6 导入器

固定流程：

1. 选择来源；
2. 识别预览；
3. 冲突检查；
4. 确认导入；
5. 执行；
6. 完成或回滚摘要。

必须显示：

- copy-only / move-only；
- 目标路径预览；
- 冲突逐项处理和批量策略；
- 不覆盖；
- 真实进度和取消；
- 单文件失败；
- 回滚结果；
- Index 写入、备份和读回状态；
- 打开目标目录；
- 操作日志。

### 11.7 设置

分组：

- 外观；
- 资源库；
- 播放和后端；
- 字幕和歌词；
- 导入偏好；
- 快捷键；
- AI 维护入口。

设置页不直接展示原始路径、IPC、日志和测试状态。

### 11.8 AI 维护

包含：

- 缺失文件结果；
- Index 备份、恢复和清理；
- Provider 缓存和错误；
- mpv / fallback 时间线；
- 导入事务和回滚；
- 维护历史；
- 版本、安装包和日志。

必须明确高风险操作的影响、预览、备份、确认和结果。

## 12. 播放器系统

三种播放器共享同一播放状态、队列、字幕、后端和持久化。

### 12.1 PlayerBar

左：封面、标题、社团/艺术家、RJ/专辑。  
中：收藏、上一首、播放暂停、下一首、模式。  
右：字幕、队列、音量、睡眠定时、更多、展开。

必须支持进度、Seek 预览、后端非阻断提示和键盘操作。

### 12.2 经典模式

定位：信息和控制平衡，适合日常使用。

包含：

- 大封面；
- 当前作品和音轨；
- 播放控制；
- 进度；
- 队列和文件树快捷入口；
- 格式、字幕和后端状态；
- 睡眠定时；
- 播放完成策略。

### 12.3 黑胶模式

定位：沉浸听歌和氛围展示。

- 封面取色只用于局部背景；
- 暗角、模糊和轻微颗粒；
- 播放时唱片缓慢旋转；
- 暂停时旋转平滑停止；
- reduced-motion 时使用静态唱片；
- ASMR 允许切换为“封面模式”，不强制黑胶视觉；
- 控制区克制，不遮挡内容。

### 12.4 歌词模式

定位：字幕阅读和长音频跟随。

- 日文、中文、双语；
- 当前行突出；
- 邻近行降低透明度和轻微模糊；
- 手动滚动后显示“回到当前行”；
- 字幕来源和文件名；
- 时间偏移；
- 无字幕和重新关联；
- SRT / VTT / ASS 使用适合段落字幕的布局；
- 桌面歌词入口。

### 12.5 队列

- 当前项不可误删；
- 拖拽显示插入位置；
- “下一首播放”和“加入队列末尾”区分；
- 单项移除有直接按钮；
- 保存队列为歌单；
- 清空有确认；
- 大队列虚拟化。

## 13. 动效系统

建议时长：

| 类型 | 时长 |
|---|---:|
| Hover / Press | 100～160ms |
| 小浮层 | 140～200ms |
| Dialog / Drawer | 180～260ms |
| 页面层级 | 180～280ms |
| PlayerBar 展开 | 260～420ms |

建议缓动：

```text
standard: cubic-bezier(0.2, 0, 0, 1)
emphasized: cubic-bezier(0.2, 0.8, 0.2, 1)
exit: cubic-bezier(0.4, 0, 1, 1)
```

规则：

- 页面切换只使用轻微透明度和 8～16px 位移；
- 播放器展开优先 shared layout；
- 歌词滚动使用跟随动画，不强制抢回用户滚动；
- 列表重排避免大范围弹簧；
- 任何动画不能延迟按钮响应；
- reduced-motion 使用淡入或立即切换。

## 14. 无障碍

必须：

- 键盘可达；
- focus-visible 清晰；
- Dialog 焦点圈定；
- Escape 和焦点返回；
- 图标按钮名称；
- 不只靠颜色表达状态；
- 文字和关键控件满足对比度；
- 动画遵循 reduced-motion；
- 点击区域足够；
- 列表和进度条提供语义；
- 屏幕阅读器不朗读隐藏工程信息。

## 15. 性能

- 大列表虚拟化；
- 封面懒加载和缓存；
- 不在渲染中反复全量过滤；
- 搜索和筛选做稳定派生；
- 维护页面按需加载；
- 黑胶和背景效果不得长期高 GPU 占用；
- 只动画 transform 和 opacity；
- 50,000 曲目基准不得明显退化；
- Skeleton 数量与可视区域匹配，不渲染完整假列表。

## 16. AI 生成规则

给设计或代码代理的提示必须包含：

- 读取本文；
- 中文桌面应用；
- 真实功能列表；
- 日常层和维护层边界；
- 当前主题和 Token；
- 页面状态：加载、空、错误、恢复；
- 键盘和 reduced-motion；
- 不生成假业务；
- 不新增未在项目范围内的功能；
- 不把页面做成等宽卡片阵列；
- 不使用蓝紫霓虹、全局渐变和泛滥玻璃；
- 输出组件、状态和交互说明，而不只输出截图。

## 17. UI PR 验收

每个 UI PR 必须提供：

- 变更页面和真实功能映射；
- 深色和浅色截图；
- 至少三档窗口 / DPI；
- Loading / Empty / Error / Recovery；
- 键盘和焦点检查；
- reduced-motion；
- 大库性能影响；
- 新增 Token 和组件说明；
- 被替代旧 UI 的删除情况；
- 自动测试和 Electron E2E 结果。

发布候选必须覆盖：

- 首页；
- 音声库网格和列表；
- RJ 详情；
- 音乐库和专辑；
- 歌单；
- 导入器全流程；
- 设置；
- AI 维护；
- PlayerBar；
- 经典、黑胶、歌词；
- 队列和搜索。

## 18. 明确禁止

- 只更换主题色；
- 所有页面使用同一种卡片布局；
- 把工程状态放在首页；
- 使用假下载、假扫描、假播放和假成功状态；
- 把绝对路径直接给 renderer；
- 在页面组件中实现文件事务或 mpv 控制；
- 混用图标风格；
- 无意义渐变、霓虹、玻璃、光晕和粒子；
- 每个页面自定义一套动画；
- 为了视觉隐藏必要错误或安全确认；
- 同时长期保留新旧两套 UI；
- 设计稿通过后不做实现对比和 Design QA。

## 19. 当前实现顺序

```text
Design Token + 基础组件
→ App Shell
→ 首页 / 首次启动 / Index 状态
→ 音声库 / RJ 详情
→ 音乐库 / 专辑 / 艺术家
→ PlayerBar / 经典 / 黑胶 / 歌词 / 队列
→ 歌单 / 历史
→ 导入器
→ 设置 / AI 维护
→ 动画、性能、无障碍和视觉回归
```

UI 实现必须与 `docs/BETA2_MASTER_PLAN.md` 和 `docs/ARCHITECTURE_QUALITY_PLAN.md` 同步推进。
