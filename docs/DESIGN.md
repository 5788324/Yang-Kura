# Yang-Kura UI Design System

> 文档状态：规则级设计基线 1.0  
> 适用范围：`0.168.0-beta.1` 之后的 UI 重构、主题扩展、组件新增和视觉验收  
> 核心原则：业务能力保留，界面分层重写；主题不是换色；动画不是装饰；任何新 UI 必须可长期维护。

## 1. 文档权威性

本文件是 Yang-Kura 后续 UI 更新的最高优先级规则，约束：

- App Shell、导航、页面布局和信息架构；
- 首页、音声库、音乐库、详情、歌单、导入器和设置；
- 经典、黑胶、歌词三种播放器；
- 深色、浅色及未来主题；
- 组件、图标、排版、动画、反馈和无障碍；
- 真实功能与 UI 操作入口的映射；
- AI、Stitch、Lovable、AppDeploy、Codex 或其他代理的实现边界；
- PR 截图和发布候选的 UI 验收。

设计稿、提示词或 UI PR 与本文件冲突时，以本文件为准。修改本文件必须说明原因、影响页面、迁移和回退方式。

## 2. 产品定位

Yang-Kura 是 Windows 本地音频媒体库，核心内容包括：

1. ASMR/RJ 音声作品；
2. 普通本地音乐；
3. 统一 Track 播放器；
4. 本地字幕与歌词；
5. 歌单、队列、历史和续播；
6. 安全导入、元数据和资源库维护。

它不是在线流媒体商城、SaaS 看板、文件管理器换皮、工程诊断控制台或只靠玻璃/渐变包装的 AI Demo。

### 2.1 体验目标

- 安静、成熟、可信；
- 封面和内容优先，控件退后；
- 大型本地库仍清晰、快速；
- 长音频续播、字幕和作品结构优先；
- 功能复杂但不暴露工程细节；
- 深色适合夜间，浅色是完整独立体验。

## 3. 核心设计原则

### 3.1 内容先于容器

- 优先展示封面、标题、音轨、字幕和播放状态。
- 不把所有信息包进等宽卡片。
- 只有需要层级、点击区域或浮层时使用 Card。
- 大列表优先使用行、分隔线、留白和列对齐。

### 3.2 日常层与维护层分离

日常层允许：首页、音声库、音乐库、歌单、导入、设置、播放器、字幕、队列、历史、搜索和用户可理解的状态。

AI 维护层集中：Index、缺失文件、备份恢复、Provider、mpv、导入事务、版本、日志和错误证据。

日常层禁止：MVP/U 编号、verifier、命令行、绝对路径、IPC、堆栈、Git SHA 和测试状态。

### 3.3 主题改变材质，不改变结构

主题可以改变：画布、表面、边框、阴影、文字、强调色、封面处理和环境背景。

主题不得改变：导航位置、按钮含义、信息顺序、键盘行为和功能可见性。

### 3.4 动画表达状态

动画只用于页面层级、浮层出现、播放器展开、布局变化、当前歌词、加载和操作反馈。

禁止持续发光、粒子、无意义漂浮、所有元素依次飞入、夸张弹跳和阻塞操作的长动画。

## 4. 信息架构

### 4.1 一级导航

固定顺序：

1. 首页；
2. 音声库；
3. 音乐库；
4. 歌单；
5. 导入资源；
6. 设置。

AI 维护不是常驻一级导航；通过设置高级入口、错误恢复入口或快捷面板进入。

### 4.2 全局能力

- `Ctrl/Cmd + K`：全局搜索；
- 常驻 PlayerBar；
- 右侧 Drawer：队列、筛选和次级详情；
- 独立 Overlay：全屏播放器；
- Toast 只提示短暂结果，不承载事务状态。

## 5. App Shell

### 5.1 桌面基准

- 主验收：1440×900；
- 最低日常：1280×720；
- 验收 Windows 100%、125%、150% DPI；
- 左侧栏常驻，底部 PlayerBar 常驻；
- 主内容区独立滚动；
- 全屏播放器覆盖应用壳但保留 Escape 和焦点返回。

### 5.2 窄窗口

- 侧栏可折叠为图标；
- 表格可横向滚动；
- 工具区允许下移；
- PlayerBar 不能隐藏主播放控制；
- 重要按钮不可被裁切。

- 深炭灰而不是纯黑；
- 石墨表面层级；
- 暖白主文字；
- 低饱和琥珀为唯一全局强调；
- 封面取色只用于播放器环境背景；

### 6.3 新主题门槛

新增主题必须提供：完整 Token、首页/资源库/详情/设置、三播放器、空/加载/错误状态、DPI 截图和对比度报告。只换 accent 不算新主题。

## 7. 排版

优先系统字体：`Segoe UI Variable`、`Microsoft YaHei UI`、`Noto Sans CJK SC`、sans-serif。

| 层级 | 大小 |
|---|---:|
| 页面主标题 | 28–32px |
| 区块标题 | 18–22px |
| 卡片/作品标题 | 14–17px |
| 正文 | 14–15px |
| 次级信息 | 12–13px |
| 微型状态 | 11–12px |

规则：

- 标题不全部加粗；
- 中文正文行高 1.55–1.7；
- 表格数字、时间、大小右对齐；
- RJ 号和技术 ID 可使用等宽字体，但不能主导视觉；
- 超长标题两行截断，完整值通过 Tooltip 或详情显示。

## 8. 间距、圆角和密度

基础间距：`4 / 8 / 12 / 16 / 20 / 24 / 32 / 40 / 48`。

- 页面外边距：24–32px；
- 区块间距：24–32px；
- 卡片内部：16–20px；
- 紧凑列表：8–12px；
- 普通按钮高度：36–40px；
- 紧凑按钮：30–32px；
- PlayerBar：72–84px。

圆角：

- 小控件 6–8px；
- 输入/按钮 8–10px；
- 卡片 12–14px；
- Drawer/Dialog 16–20px；
- 禁止全局药丸化。

## 9. 图标

统一使用 Lucide 或同等风格线性 SVG。禁止混用 Emoji、Unicode 和多套图标库。

- 导航：20px；
- 工具栏：18px；
- 紧凑按钮：16px；
- 默认描边 1.75；
- 选中态 2；
- 仅图标按钮必须有 Tooltip 和 aria-label。

## 10. Motion 系统

推荐 Motion/Framer Motion 类方案，参数从统一文件读取。

| 场景 | 建议 |
|---|---|
| Hover | 100–140ms |
| 按压 | 80–120ms |
| 页面切换 | 180–240ms |
| Drawer/Dialog | 220–300ms |
| PlayerBar→全屏 | 320–420ms shared layout |
| Toast | 180–240ms |

原则：

- 页面切换只淡入和 8–16px 位移；
- Drawer 使用轻弹簧，不反复弹跳；
- 黑胶只有播放时旋转；
- 歌词使用位移、透明度和有限模糊；
- reduced-motion 下关闭 shared layout、连续旋转和大位移，只保留短淡入。

## 11. 基础组件

### 11.1 Button

类型：Primary、Secondary、Ghost、Danger、Icon。

必须有 hover、active、focus、disabled、loading。一个局部区域最多一个 Primary。

### 11.2 Media Card

用于作品/专辑；必须显示封面、主标题、核心元数据、播放/字幕状态。禁止默认显示工程字段。

### 11.3 Track Row

必须支持序号/播放态、标题、时长、字幕、进度、More 菜单；播放控制不能只在 Hover 出现。

### 11.4 Drawer

用于队列、筛选和次级信息；必须支持 Escape、焦点圈定、关闭后焦点返回。

### 11.5 Dialog

用于确认、字段比较和危险操作；危险操作必须明确对象、范围和结果，不使用模糊文案。

### 11.6 Empty State

必须说明发生了什么、为何为空、下一步主操作和可选次操作。禁止假数据填充。

### 11.7 Feedback

- Toast：短暂结果；
- Inline Alert：局部可恢复问题；
- Banner：页面级阻断或授权失效；
- Progress Panel：导入/扫描/恢复等长事务。

## 12. 页面规范

### 12.1 首页

首屏优先：继续播放、未听完、最近加入、资源库状态和快捷入口。

必须设计：首次启动空状态、Index 损坏恢复、新文件待扫描、后端 fallback 提示。

### 12.2 音声库

必须支持网格/列表、排序、播放/字幕/文件健康筛选、多选、批量收藏/标签/歌单和密度切换。

### 12.3 RJ 详情

必须覆盖：播放、收藏、编辑资料、DLsite 匹配、文件夹、图片/文本附件、元数据备份恢复、音轨、字幕和文件健康。

元数据 Provider 必须字段级对比和选择性应用，禁止一键静默覆盖。

### 12.4 音乐库

提供专辑、艺术家、音轨；专辑详情使用音轨表格；支持标签、封面、格式和缺失信息状态。

### 12.5 歌单、队列和历史

歌单支持新建、改名、封面、说明、删除。队列支持拖拽、移除、下一首播放、保存歌单。历史支持单条和时间范围清理确认。

### 12.6 导入器

固定流程：来源 → 识别 → 目标/冲突 → 确认 → 执行 → 完成。

必须覆盖扫描进度、取消、目标树、逐项冲突、批量策略、单项失败、回滚结果、Index 写入/读回验证和完成摘要。

### 12.7 设置

按外观、资源库、播放、字幕、导入、高级分区。设置页不显示日志、verifier、绝对路径和 Git 信息。

### 12.8 AI 维护

提供缺失文件、Index 备份恢复、Provider、mpv/fallback、导入事务、版本和日志。危险操作进入独立 Danger Zone。

## 13. 三种播放器

### 13.1 经典

定位：高效率日常播放。

必须包含封面、标题、作者/社团、上一首/播放/下一首、进度、音量、队列、字幕、收藏、播放模式和展开。

### 13.2 黑胶

定位：沉浸体验。

使用封面取色环境背景、唱片和克制控制。ASMR 可切换封面模式，不强制黑胶隐喻。暂停时停止旋转并保持角度。

### 13.3 歌词

定位：字幕阅读。

支持日文、中文、双语；当前行突出，上下文弱化；支持偏移、回到当前行、来源/文件名、重新关联和无字幕状态。

## 14. 真实功能映射

正式 UI 必须保留以下能力入口：

- 目录授权与扫描；
- Index 写入、读取、备份和恢复；
- 音声库/音乐库/歌单/历史；
- HTMLAudio/mpv/fallback；
- LRC/SRT/VTT/ASS；
- 外部打开；
- copy/move 导入和回滚；
- 本地元数据和 DLsite Provider；
- 缺失文件、索引清理和维护历史；
- portable/NSIS 版本信息仅在 AI 维护显示。

## 15. 状态模型

每个异步操作至少定义：idle、loading、success、empty、warning、error、permission-required、retrying、cancelled。

文件事务额外定义：preview-ready、awaiting-confirmation、executing、rolling-back、rolled-back、partial-failure、readback-verifying、completed。

禁止用一个 Toast 代替完整事务状态页。

## 16. 无障碍和键盘

- 明显 focus-visible；
- Dialog/Drawer/全屏播放器焦点圈定；
- Escape 关闭并恢复焦点；
- 图标按钮 aria-label；
- 主题满足 WCAG AA；
- 当前播放/选中不能只靠颜色；
- Space 播放暂停；Ctrl/Cmd+K 搜索；M 静音；Q 队列；L 歌词模式。

## 17. 性能规则

- 作品、音轨和搜索使用虚拟列表或分页；
- 封面懒加载；
- 不在列表 render 中做复杂计算；
- 封面取色缓存；
- 页面切换不重新扫描文件系统；
- 动画不阻塞扫描、导入和播放事件；
- 50,000 Track 场景必须保持可操作。

## 18. 工程结构建议

```text
src/
├─ app/
├─ design-system/
│  ├─ tokens/
│  ├─ themes/
│  ├─ motion/
│  ├─ primitives/
│  └─ patterns/
├─ features/
│  ├─ home/
│  ├─ library/
│  ├─ player/
│  ├─ playlist/
│  ├─ importer/
│  ├─ metadata/
│  ├─ subtitles/
│  └─ settings/
└─ shared/
```

页面不直接写颜色，不直接操作文件，不直接控制 mpv，不创建私有主题变量；Motion 参数从统一文件读取。

## 19. AI 生成要求

任何 AI UI 任务必须包含：产品定位、目标页面、真实功能、允许范围、必须保留状态、主题、禁用模式、窗口/DPI、键盘/reduced-motion 和验收截图。

禁止提示：

> 做一个高端、科技感、炫酷、玻璃拟态的音乐播放器。

推荐提示：

> 按 `docs/DESIGN.md` 重写音声库列表，保留 RJ、社团、声优、标签、字幕、进度、排序、筛选、多选和 More 菜单；目标 1440×900 与 1280×720；不得新增功能或等宽 Dashboard 卡片。

## 20. PR 验收清单

每个 UI PR 必须回答：

- 修改页面/组件；
- 使用的 Token；
- 新增状态；
- 深浅主题；
- 1280×720；
- 三档 DPI；
- 键盘、Escape、焦点返回；
- reduced-motion；
- 空/加载/错误/权限状态；
- 是否暴露工程细节；
- 50,000 Track 是否退化；
- 截图和自动测试证据。

最低截图：深浅首页、深浅资源库、详情、设置、经典/黑胶/歌词、队列、导入冲突、AI 维护。

## 21. 明确禁止

- 纯黑大面积背景；
- 紫蓝霓虹 AI 渐变；
- 大面积毛玻璃；
- 等宽三卡片作为所有页面默认布局；
- Emoji 图标；
- 主题只换 accent；
- 假统计、假下载、假扫描和假成功；
- 日常首页显示工程状态；
- Renderer 显示绝对路径；
- 未确认的移动、覆盖、清理和恢复；
- 为视觉效果牺牲大库性能；
- 在稳定版中为冻结功能加入可点击假入口。

## 22. 版本治理

- Major：信息架构、主题模型或核心交互规则变化；
- Minor：新页面规范、组件模式或主题；
- Patch：文案、数值、示例和澄清。

当前版本：`1.0.0`。
