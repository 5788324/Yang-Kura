# MVP-41 — 播放器详情页 / 歌词页体验收口

## 目标

MVP-41 的目标是把播放器详情页和歌词页从“功能演示面板”继续收口成日常听音频的媒体产品界面。

本轮不改变播放内核，不改变索引格式，不读取新的本地文件，也不接入任何联网元数据。

## 新增服务

```text
src/services/playerSurfaceExperienceService.ts
```

该服务集中生成播放器表层展示信息：

- 当前模式标题
- 当前模式说明
- 当前播放来源提示
- 当前音轨上下文
- 歌词 / 字幕提示
- 播放页底部状态
- 播放状态标签
- 播放统计卡片

## 用户可见效果

### 播放器详情页

- 顶部显示“播放详情 / 黑胶沉浸 / 歌词专注”等用户可理解的模式标题。
- 当前音轨信息下方显示统一状态标签。
- 详情页显示总时长、播放进度、队列数量、歌词行数。
- 来源信息从“当前播放”等泛化文本改为“本地音频播放 / 字幕状态”等真实用户语义。

### 歌词页

- 无歌词时显示更具体的提示：字幕读取中、暂无字幕、读取失败、发现字幕候选等。
- 底部状态改为用户可见的播放状态汇总。
- 分轨、队列、片段标记等文本继续中文化，减少原型/工程味。

### 底部播放器栏

- “桌面悬浮歌词”改成“歌词浮窗”，避免暗示已经实现系统级桌面歌词能力。

## 安全边界

本轮只做前端展示和文案收口。

继续禁止：

```text
不接 SQLite
不接下载器
不抓 ASMR.one / DLsite 元数据
不接 mpv
不删除文件
不移动文件
不重命名文件
不把 absolutePath / file:// 暴露给 renderer
不保存 mediaUrl / resolvedMediaUrl 到新的持久化结构
```
