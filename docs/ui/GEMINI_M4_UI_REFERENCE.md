# Gemini M4 UI Reference

This document clarifies the relationship between Gemini's M4 UI visual prototype
and the actual Python Flet M4 implementation.

## 1. Gemini UI as Visual Reference Only

The React/Vite UI generated by Gemini / AI Studio is a **visual prototype**.
It is NOT imported into this project.  The M4 Flet UI is built from scratch
in Python using the visual ideas but NOT the React code.

## 2. Current M4 Scope

M4.0 only implements the **Library management UI**.

| module | status |
|---|---|
| Library / 资源库 | Active |
| Downloader / 下载器 | Coming Soon |
| Player / 播放器 | Coming Soon |
| Settings / 设置 | Placeholder |
| Diagnostics / 诊断 | Placeholder |

## 3. NOT Implemented (M4.0)

The following features from the Gemini mockup are explicitly NOT implemented:

| feature | reason |
|---|---|
| Download progress bars | downloader is not connected yet |
| Play buttons that play audio | player is Coming Soon |
| DLsite metadata crawling | M6+ task, requires network |
| Cloudflare bypass UI | not applicable to local manager |
| aria2c log display | downloader Coming Soon |
| Delete / rename resource files | blocked by project safety rules |
| Metadata refresh via network | M6+ task |
| Subtitle cloud matching | not in scope |
| Fake file writes / mock data updates | data comes from real SQLite DB |

## 4. Long-Term Navigation Placeholders

The sidebar shows these entries so the UI layout is familiar early:

- **Library / 资源库**: lands on current working page
- **Downloader / 下载器**: greyed out, Coming Soon tooltip
- **Player / 播放器**: greyed out, Coming Soon tooltip
- **Settings / 设置**: basic placeholder view

## 5. Data Source

All M4 data comes from the **real** SQLite DB at:

- `data/yang_kura_real.db` (default)
- or the path in env var `YANG_KURA_DB_PATH`

No mock data.  No fixture data.  The UI reads works, media_files, and
scan_runs exactly as stored.
