# MVP121 元数据来源管理收口

## 目标

单个 RJ 的 Provider 候选信息不再只能整批填入。用户可以逐项选择字段，比较本地当前值和外部候选值，再填入编辑表单并手动保存。

## 用户流程

```text
查询 RJ
→ 本地当前值 / 外部候选值并排比较
→ 字段逐项选择
→ 填入已选字段
→ 用户确认保存
→ 本地覆盖记录保存来源标识
```

支持字段：标题、社团、声优、发售日期、简介、标签。

## 来源标识

本地覆盖记录可保存：

- 手动编辑或 Provider 候选来源；
- Provider 名称和来源标签；
- 外部页面 URL（只接受 HTTP/HTTPS）；
- Provider 抓取时间；
- 实际采用的字段；
- 本地保存时间。

历史没有来源信息的覆盖记录显示为“本地手动修改（旧记录）”。来源信息不计入“本地修改字段数”。

## 可重复成功路径

`tests/fixtures/dlsite/RJ01554928-translation-success.html` 是合成离线夹具，用于验证：

- 请求 RJ 为 `RJ01554928`；
- 最终页面可为 `RJ01554927?translation=RJ01554928`；
- 解析结果仍归属请求的 `RJ01554928`；
- 标题、社团、声优、日期、简介和标签均可进入候选模型。

夹具只验证程序路径，不冒充真实作品内容。

## 安全边界

- 不自动保存 Provider 候选；
- 不自动覆盖本地修改；
- 不修改音频文件标签；
- 不写媒体目录；
- 不保存 absolutePath 或 file://；
- 不做全库抓取；
- 不接 SQLite。
