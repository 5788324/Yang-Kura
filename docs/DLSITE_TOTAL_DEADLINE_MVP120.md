# MVP120：DLsite 单次查询总截止时间

Codex 在 Windows 实网复验中发现，MVP119 对 6 个候选 URL 分别使用完整 12 秒超时，网络不可达时理论耗时可扩大到约 72 秒。

MVP120 改为一次查询只创建一个总截止时间：

1. 默认总预算仍为 12 秒。
2. 总截止时间在候选 URL 循环之前创建一次。
3. 每个候选 URL 只能使用剩余时间。
4. 剩余时间耗尽后立即返回 `mvp119-dlsite-timeout`。
5. 页面所说的“12 秒”现在对应整个单 RJ 查询，而不是每个候选 URL。

本轮不改变缓存、5 秒节流、域名白名单、差异预览、手动保存或 JSON 回退。
