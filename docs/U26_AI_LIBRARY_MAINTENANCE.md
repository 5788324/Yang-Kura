# U26 资源库检修集中到 AI 维护

## 目标

继续执行日常界面硬规则，把资源库页面中低频、高风险或排障性质的操作集中到默认折叠的 `AI 维护 · 资源库检修`，避免普通设置页长期展示大量检修窗口。

## 默认可见

资源库设置页继续直接展示：

- 选择音声库和音乐库目录；
- 上次资源库的集合、音轨和读取时间；
- 读取已有资源库记录；
- 一键扫描并应用；
- 音声库和音乐库路径列表与基础增删操作。

## AI 维护

以下能力继续保留，但放入原生 `details` 默认折叠区：

- 缺失文件与失效记录检查；
- 重新定位和只读重新扫描；
- 受控索引移除预览与确认写入；
- 索引备份列表、手动恢复和保留策略预览；
- 最近维护历史；
- 扫描预览、写入预览和高级资源库工具。

## 历史兼容

- `mvp127 / mvp128 / mvp129 / mvp39` ID 和全部处理函数继续保留；
- 历史回归卡 `mvp54-settings-regression-path` 继续作为隐藏兼容标记，不在维护区重复展示；
- 不删除 Electron IPC、Local JSON Index、备份、恢复、清理或扫描能力。

## 交互边界

- `AI 维护 · 资源库检修` 默认关闭，不使用持久化状态；
- 用户主动展开后，现有按钮、确认文本、结果和错误提示行为保持不变；
- 内部“高级资源库工具”仍可二次展开，避免一次呈现全部扫描与写入预览；
- 不修改资源库数据、路径 token、播放器、导入器、下载器或诊断页。

## 验证

U26 专项门禁必须同时验证默认折叠、检修能力完整包裹、日常路径管理位于折叠区之外、历史回归卡保持隐藏，以及 SettingsPage 的 TypeScript 转译无错误。
