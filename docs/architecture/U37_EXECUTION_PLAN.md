## 目标

U37 不新增媒体能力，而是把现有首页、音声库、RJ 详情、音乐库及其钻取页面迁移到统一 Design System，并在不改变 Index、播放、字幕、元数据和导入事务行为的前提下补齐日常使用状态。

U37 已完成，项目现在直接进入个人日用版 Windows 发布候选，不等待其他大型 UI 或结构阶段。

跟踪：

- [Issue #65：完成媒体库并发布个人日用版](https://github.com/5788324/Yang-Kura/issues/65)
- [Issue #66：渐进式结构治理与质量提升](https://github.com/5788324/Yang-Kura/issues/66)

## 固定原则

- 现有 `AudioTrack`、`RJWork`、`MusicAlbum` 和 `LibrarySessionSnapshot` 数据模型保持兼容。
- 不修改 Electron 文件事务、rootPathToken、相对路径或媒体协议。
- 页面迁移同时覆盖空状态、无效选择、渲染错误、键盘焦点和小窗口。
- 优先复用 `Button`、`Surface`、`Feedback`、`MediaCard`、`TrackRow`。
- 每个子轮次保持可运行，不留下依赖下一轮才能恢复的中间态。
- 修改页面时同步整理该页面触碰的高耦合职责，不进行全项目推倒重写。
- 小型、低风险且相关的修改合并处理，完整门禁集中在 PR 收口时执行。

## 子轮次

### U37-A：页面状态与错误恢复 — 已完成

- `LibraryPageState` 统一页面连接状态分类。
- `LibraryRouteBoundary` 把渲染异常限制在当前资源库页面，并提供原地重试。
- AppRouter 对首页、音声库、RJ 详情和音乐库建立页面边界。
- RJ 详情 ID 失效时提供明确返回入口。

### U37-B：首页与音声库列表 — 已完成

- 新增正式 `HomeLibraryPage` 和 `AsmrLibraryPage`。
- 迁移首页、搜索、排序、社团、声优、标签、来源、字幕、播放进度和个人状态筛选。
- 网格与列表分别使用共享 `MediaCard`、`TrackRow`，并支持键盘激活。
- 加入作品多选、全选当前结果和批量加入歌单。
- 保留 50,000 曲目窗口化策略和现有搜索索引。

### U37-C：RJ 详情 — 已完成

- 新增正式 `RjDetailPage`，迁移详情 Hero、播放入口、作品信息、字幕覆盖和资源健康状态。
- 音轨列表使用共享 `TrackRow`，支持播放、队列、收藏、外部打开和文件管理器定位。
- 新增 `RjMetadataDialog`，保留本地覆盖、清除覆盖、DLsite 查询、差异预览和字段选择性应用。
- 评分、个人听音状态和笔记保存到本地元数据覆盖层，不修改媒体文件。
- 无音轨、无字幕、资源警告和 Provider 失败具备明确恢复或解释状态。

### U37-D：音乐库、专辑与艺术家详情 — 已完成

- 新增正式 `MusicLibraryPage`，替换旧音乐库生产路由。
- 歌曲、专辑、艺术家、文件夹四种视图使用统一工具栏和页面状态。
- 专辑、艺术家和文件夹支持钻取详情、返回导航、播放全部和全部加入队列。
- 专辑墙与分组使用共享 `MediaCard`，曲目使用共享 `TrackRow`。
- 支持搜索、排序、仅看收藏、多选、全选当前结果和批量加入队列。
- 保留播放、队列、收藏、外部打开、文件定位、音乐元数据覆盖和大库渲染窗口。
- 新增语义主题样式、窄窗口响应布局、键盘激活和 reduced-motion。
- 删除旧 `src/components/MusicLibrary.tsx`，不保留长期双轨。
- U32 Electron 审计验证四视图、钻取详情、批量队列、收藏筛选和窄窗口。
