# U37 资源库与详情 UI 执行计划

## 目标

U37 不新增媒体能力，而是把现有首页、音声库、RJ 详情、音乐库及其钻取页面迁移到统一 Design System，并在不改变 Index、播放、字幕、元数据和导入事务行为的前提下补齐日常使用状态。

## 固定原则

- 现有 `AudioTrack`、`RJWork`、`MusicAlbum` 和 `LibrarySessionSnapshot` 数据模型保持兼容。
- 不修改 Electron 文件事务、rootPathToken、相对路径或媒体协议。
- 页面迁移必须同时覆盖空状态、无效选择、渲染错误、键盘焦点和小窗口。
- 优先复用 `Button`、`Surface`、`Feedback`、`MediaCard`、`TrackRow`，禁止继续复制页面级颜色常量。
- 每个子轮次保持可运行，不留下必须依赖下一轮才能恢复的中间态。

## 子轮次

### U37-A：页面状态与错误恢复 — 已完成

- `LibraryPageState` 统一页面连接状态分类。
- `LibraryRouteBoundary` 把渲染异常限制在当前资源库页面，并提供原地重试。
- AppRouter 对首页、音声库、RJ 详情和音乐库建立页面边界。
- RJ 详情 ID 失效时提供明确返回入口，不再出现空白内容区。
- 建立 U37 页面容器和状态视觉样式。

### U37-B：首页与音声库列表 — 已完成

- 新增正式 `HomeLibraryPage`，统一资源库状态、继续播放、最近播放、常用入口、最近加入和歌单入口。
- 新增正式 `AsmrLibraryPage`，迁移搜索、排序、社团、声优、标签、来源、字幕、播放进度和个人状态筛选。
- 网格与列表分别使用共享 `MediaCard`、`TrackRow`，并支持键盘激活。
- 加入作品多选、全选当前结果和批量加入歌单等低风险操作。
- 保留 50,000 曲目窗口化策略和现有搜索索引。
- 单作品编辑、刷新、喜欢和非破坏性移除保持可用。
- U32 Electron 审计直接验证正式页面、多选、歌单持久化和视图切换。

### U37-C：RJ 详情 — 已完成

- 新增正式 `RjDetailPage`，重排 Hero、作品元数据、播放入口、字幕覆盖和资源健康信息。
- 音轨列表迁移到共享 `TrackRow`，支持应用内播放、加入队列、收藏、系统外部打开和文件管理器定位。
- 新增正式 `RjMetadataDialog`，保留本地覆盖、清除覆盖、DLsite 查询、字段差异预览和选择性应用。
- 评分、个人状态和笔记统一写入本地元数据覆盖层，不修改媒体文件。
- 无音轨、无字幕、资源警告和 Provider 失败提供明确恢复或解释状态。
- U32 Electron 审计直接验证详情导航、TrackRow、个人状态持久化、元数据弹窗和 Escape 关闭。

### U37-D：音乐库、专辑与艺术家详情 — 当前任务

- 歌曲、专辑、艺术家、文件夹四种视图统一页面工具栏和钻取导航。
- 专辑/艺术家详情与音轨列表使用共享 MediaCard/TrackRow。
- 补齐多选、批量加入队列、收藏筛选和外部打开反馈。
- 完成 U37 窗口、主题、键盘、空状态和 Electron 回归矩阵。

## U37 完成条件

- 首页、音声库、RJ 详情、音乐库、专辑/艺术家钻取均使用正式语义样式。
- 未连接、空 Index、无结果、无效详情 ID 和渲染错误均有明确恢复路径。
- 主要列表具备多选与批量操作，且不破坏播放队列或元数据覆盖。
- U28～U32、稳定回归、portable 与 NSIS 继续通过。
- 完成后进入 U38 播放器与歌词纵向迁移。
