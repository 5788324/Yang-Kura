# Yang-Kura MVP126 当前路线

版本：`0.164.0-mvp126`

## 本轮目标

在不读取真实大资源库的前提下，建立大库性能基准并降低日常页面首屏负载。

## 已完成

- 生成 4,000 个音声作品、1,500 个音乐专辑、50,000 条曲目的内存测试夹具。
- 音声库搜索使用预计算搜索文本和 `useDeferredValue`。
- 音乐库搜索使用专辑/曲目搜索索引，艺术家统计改为一次聚合。
- 音声作品、音乐曲目、专辑、艺术家和文件夹分批渲染。
- Dashboard、音声库、音声详情、音乐库、歌单和诊断入口按路由拆包。
- 完整诊断页改为二阶段按需加载。
- React 与 Lucide 图标拆为独立 vendor chunk。

## 明确未做

- 不扫描真实大库。
- 不修改 `library-index.json`。
- 不复制、移动、删除或重命名媒体文件。
- 不接 SQLite。
- 不改变播放器、导入器和 Provider 的安全边界。

## 下一阶段建议

MVP127 应转向实际用户收益更高的功能，不继续做纯性能微调。优先选择：

1. 缺失文件与失效索引管理；或
2. 批量本地元数据编辑；或
3. 下载任务模型与 staging 目录基础。
