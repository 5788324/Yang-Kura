# 下一阶段计划

## 当前阶段：真实日常使用观察

MVP129 已稳定合入。下一阶段不是继续堆新模块，而是使用实际资源库观察 1～3 轮，只处理明确问题。

优先观察：

1. 真实资源库浏览、搜索、筛选和页面跳转。
2. 2～10 小时音频的播放、Seek、暂停、切歌和进度恢复。
3. 字幕显示、播放器退出、mpv fallback 和残留进程。
4. 小样本 copy-only；确有需要时再做小样本 move-only。
5. 元数据编辑、DLsite 单 RJ Provider、索引清理与备份恢复交互。

## 只有明确痛点后才选择一个方向

| 候选方向 | 启动条件 |
|---|---|
| 播放稳定性与设备设置 | 实际使用发现 mpv/HTMLAudio 问题 |
| 批量元数据 | 单项编辑稳定且存在批量需求 |
| 字幕/本地转录衔接 | RJ-LRC-Local 已准备接入 |
| SQLite 评估 | JSON Index 出现真实性能或一致性瓶颈 |
| 系统媒体控制 | 播放主链长期稳定后 |
| 下载器 | 用户明确恢复 MVP130 路线后 |

禁止并行展开多个大方向。每次只做一个可验证的小闭环。
