## 阶段 A：稳定包与 Git 合入（当前）

1. 精简未使用依赖和重复构建工具。
2. 归档 MVP01～MVP111 历史快照与交接资料。
3. 统一接手文档到 MVP129。
4. 跑 `verify:stable`、构建和审计。
5. 在 Windows 再跑 portable / installer 发布门禁。
6. Codex 提交并推送 GitHub，核对远端 HEAD。

## 阶段 B：真实日常使用观察（优先）

预计 1～3 轮，仅处理明确问题：

- 真实资源库浏览、搜索和页面操作。
- 长音频播放、Seek、暂停、切歌和进度恢复。
- 实际字幕显示和播放退出。
- 小样本 copy-only；必要时小样本 move-only。
- 元数据编辑、Provider 和索引维护的真实交互。

禁止在没有真实痛点时继续拆微功能。

## 阶段 C：候选增强方向

按实际需求选择一个，不并行展开：

| 方向 | 条件 |
|---|---|
| mpv 长期稳定性/设备设置 | 真实使用发现播放问题时 |
| 批量元数据 | 单项编辑已经稳定且确有批量需求时 |
| 字幕与本地转录衔接 | RJ-LRC-Local 工作流准备接入时 |
| SQLite 评估 | JSON Index 在真实大库中出现性能或一致性瓶颈时 |
| 系统媒体控制 | 播放主链稳定后 |
