# PROJECT_ROADMAP

> **文档定位：Yang-Kura 唯一长期开发路线真源。**
>
> 所有新功能、结构调整、验收和发布任务必须先与本文件对齐。除非用户明确改变方向，否则后续 AI、Codex 和其他执行者不得绕过本路线自行启动大功能。

## 1. 文档职责与优先级

项目文档按以下职责使用：

1. **GitHub `main`**：唯一代码事实来源。
2. **`PROJECT_STATE.md`**：当前版本、已完成事实和当前任务快照。
3. **`PROJECT_ROADMAP.md`**：长期阶段、执行顺序、轮次预算、冻结项和启动条件。
4. **`docs/U*.md`**：单轮实现、边界和验收记录。
5. **`docs/UI_DAILY_SURFACE_RULES.md`**：长期 UI 硬规则。

不得在长期文档中固定某个 Git SHA。执行任务时应从 `origin/main` 获取当前 HEAD 并记录在当次报告中。

## 3. 一轮任务的定义

本路线中的“一轮”默认指：

```text
一个明确且有限的任务范围
→ 独立分支
→ 必要实现或验收
→ 专项检查
→ 完整稳定回归
→ Windows 构建门禁
→ 审查后合入 main
```

简单、低风险、互相关联的事项可以在同一轮合并处理；高风险文件操作、播放后端、数据迁移和安装发布不得与无关任务捆绑。

## 5. 当前唯一主线：U27～U33 新 Beta 发布

### U27：Windows GUI 验收基线（已完成）

**预计：1 轮。原则上不开发新功能。**

**实机结果：CONDITIONAL GO。** 0 Blocker、0 Major、1 Minor、2 Observation；自动门禁、strict smoke、portable、空库 UI、三主题和进程回收通过。原生目录选择器受自动化限制，扫描、真实播放/Seek、字幕和导入保持 NOT TESTED，转入 U28 人工补测。详细结果见 `docs/U27_WINDOWS_GUI_ACCEPTANCE_RESULT.md`。

目标：

- 从最新 `origin/main` 建立可重复的 Windows 实机验收基线。
- 固定测试环境、测试样本、候选 portable 包和问题分级方式。
- 验证当前自动门禁之外的真实窗口、真实点击和进程行为。

必须覆盖：

- 干净首次启动。
- 临时 ASMR 与音乐样本库。
- 中文、日文、空格、特殊字符和较长路径。
- 长音频、多轨作品、无封面、无字幕和四种字幕格式。
- 三主题、常规窗口、小窗口和最大化。
- AI 维护默认折叠。

产出：

- Windows GUI 验收报告。
- 截图和日志索引。
- Blocker / Major / Minor / Observation 问题表。
- 是否进入 U28 的结论。

Codex 执行任务见：`docs/U27_WINDOWS_GUI_ACCEPTANCE_TASK.md`。

### U28：原生目录选择与真实媒体闭环（当前）

**预计：1 轮。原则上不开发新功能。**

U28 是 U27 `CONDITIONAL GO` 的补充实机轮，第一道门是通过 Windows 原生目录选择器人工授权仓库外临时样本。不得用脚本、配置文件或开发者工具注入绝对路径绕过 GUI。

必须完成：

2. 扫描、写入、备份、读取并应用 Local JSON Index。
3. 音声库和音乐库显示真实临时媒体。
4. HTMLAudio 真实播放、点击/拖拽/长音频 Seek、队列与续播。

8. move-only 仅使用独立临时副本，验证二次确认、不覆盖、不清理源目录和 OperationLog。
9. 重启后的资源库、播放历史和字幕关联恢复。

11. 复核 MIN-001：导入器未选来源时“4 个示例文件”是否仍可能被理解为当前识别结果。

用户只在原生目录对话框中完成目录选择，Codex 继续执行其余 GUI 操作、检查和报告。详细任务见 `docs/U28_NATIVE_LIBRARY_WORKFLOW_TASK.md`。

## 7. Beta 后的大功能路线

Beta 发布后，每次只能选择一个大功能域进入主线。不得并行启动两个大功能。

### 方向 A：MVP130 正式下载器

**预计：基础可用 6～8 轮；完整队列版 9～12 轮。**

推荐顺序：

```text
重新审计 MVP130 独立实验
→ DownloadTask 与 Manifest
→ Staging 暂存区
→ 单任务下载
→ 暂停/继续/取消/重试
→ HTTP Range 断点续传
→ 应用重启恢复
→ Provider 合同
→ 单一 RJ Provider
→ 下载完成后交给 Importer
```

硬边界：

- 下载器只能写 Staging。
- 不得自动写正式媒体库。
- 不得绕过 DRM、验证码或访问控制。
- 下载结果必须经过 Importer 预览和确认。

### 方向 B：AI 维护助手

**预计：只读 MVP 4～6 轮；受控实用版 8～12 轮；完整助手 13～20 轮。**

阶段：

1. 只读：版本、资源库、日志、播放器错误和缺失文件解释。
2. 受控操作：重新扫描、生成清理预览、备份、恢复和重新定位。
3. 聊天与工具：OpenAI-compatible、本地模型、流式输出、工具调用和失败恢复。

所有写操作必须预览、确认、记录和可回退。

### 方向 C：RJ 转录、翻译和字幕工作流

**预计：外部 Worker 接入 5～7 轮；任务队列与断点续跑 8～12 轮；强制对齐和模型管理 12～18 轮。**

推荐采用独立 Worker：

```text
Yang-Kura 创建任务
→ Electron main 启动 Arsm_Transcribe / RJ-LRC-Local Worker
→ Worker 通过磁盘 JSON 和 Manifest 汇报状态
→ Yang-Kura 关联 ja / zh / bilingual LRC
```

不得把 Python ASR、翻译模型和 GUI 直接塞进 Renderer。

### 方向 D：元数据中心

**预计：5～10 轮。**

能力候选：

- DLsite 官方数据。
- ASMR.one 辅助数据。
- 本地文件夹、文本和音频标签。
- 用户手动覆盖。
- 多来源字段对比。
- 选择性应用和恢复。
- 封面缓存。
- 标签别名与中文化。

优先级：

```text
用户手动覆盖
> 本地真实文件
> DLsite 官方
> ASMR.one
> 自动推断
```

禁止全库无确认自动覆盖。

### 方向 E：OpenList / WebDAV 远程资源库

**预计：只读浏览 5～8 轮；播放与缓存 8～12 轮；完整 WebDAV 10～16 轮。**

必须单独解决鉴权、Range、缓存、断线恢复、字幕、封面和凭证保存。不能与下载器同轮启动。

### 条件方向 F：SQLite

**预计：8～14 轮。**

只有出现以下真实瓶颈时启动：

- 十万级资源导致启动、搜索或写入明显变慢。
- 复杂查询和多关系数据显著增加。
- Local JSON Index 产生一致性、并发或事务问题。
- 下载任务、转录任务或操作日志需要可靠事务。

不得为了“看起来专业”提前全面迁移。媒体库 Index 可以与任务 SQLite 并存。

### 条件方向 G：Player Core v2

**预计：10～16 轮。**

只有明确需要以下能力时启动：

- 播放速度。
- A-B 循环。
- 输出设备选择。
- 系统媒体控制。
- 更复杂的后端能力协商。
- 无缝播放、ReplayGain 或 DSP。

内核继续采用 mpv + HTMLAudio fallback，不重新自研解码器。

## 9. 质量与安全硬规则

1. 所有任务从最新 `origin/main` 创建独立分支。
2. 不在 `main` 上直接试验。
3. 不修改真实用户媒体进行破坏性测试。
4. copy-only 优先；move-only 只能使用临时副本并要求二次确认。
5. 播放、导入、Index、备份、恢复和安装链必须保留稳定回归。
6. 日常 UI 不显示 MVP、verifier、IPC、Git、命令行或原始工程状态。
7. 绝对路径和 `file://` 不进入 Renderer 日常界面。
8. 不为了代码整齐进行全项目重构。
9. 新 verifier 优先验证行为和稳定合同，不绑定局部变量名或实现文件位置。
10. 三主题和窗口布局以 Windows 实机结果为准。
11. high / critical 依赖风险阻止合并；moderate 风险显式记录，不自动升级。
12. 最终发布前必须通过 portable、NSIS、安装、卸载、进程回收和 SHA-256 验收。

## 10. 路线变更程序

改变本路线前必须：

1. 暂停开发。
2. 审计当前 `main`、开放 PR 和项目状态。
3. 明确新方向的用户价值、风险和预计轮次。
4. 说明哪些冻结项被解除。
5. 先更新 `PROJECT_ROADMAP.md` 和 `PROJECT_STATE.md`。
6. 再创建执行分支。

未经用户明确决策，不得自行把候选增强方向升级为当前任务。
