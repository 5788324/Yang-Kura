# PROJECT_ROADMAP

> **文档定位：Yang-Kura 唯一长期开发路线真源。**
>
> 所有新功能、结构调整、验收和发布任务必须先与本文件对齐。除非用户明确改变方向，否则后续 AI、Codex 和其他执行者不得绕过本路线自行启动大功能。

## 1. 文档职责与优先级

1. **GitHub `main`**：唯一代码事实来源。
2. **`PROJECT_STATE.md`**：当前版本、已完成事实、阻断项和当前任务快照。
3. **`PROJECT_ROADMAP.md`**：长期阶段、执行顺序、轮次预算、冻结项和启动条件。
4. **`docs/U*.md`**：每轮实现、验收、问题和交接记录。
5. **`docs/UI_DAILY_SURFACE_RULES.md`**：长期 UI 硬规则。

执行任务时必须从最新 `origin/main` 开始并在当次报告中记录实际 HEAD。

## 3. 一轮任务的定义

```text
一个明确且有限的任务范围
→ 独立分支
→ 实现或验收
→ 专项检查
→ 完整稳定回归
→ Windows 构建门禁
→ 审查后合入 main
```

高风险文件操作、播放后端、数据迁移和安装发布不得与无关任务捆绑。

## 5. U27 最终验收结论

### 已通过

- 自动门禁、TypeScript、Electron 编译、稳定回归和生产构建。
- strict smoke、portable 打包和进程回收。
- 干净首次启动、空库 UI、AI 维护默认折叠、三主题和常规/最大化窗口。
- 用户原配置恢复，Git 工作区 clean。

### 真实资源补测发现

#### MAJ-001：资源库授权与 Index 状态断裂

选择真实 `E:\arsm` 后，页面显示已选择目录，顶栏仍显示 51 条音轨，但设置页仍要求选择目录，读取/扫描按钮 disabled，音声库与浏览页为 0。

#### MAJ-002：诊断页仍使用 Demo 状态

AI 维护中的刷新资源状态明确执行 Demo 扫描并声明不读取真实磁盘，无法反映已授权目录或真实 Index。

### 最终结论

```text
NO-GO
```

先前空库阶段的 `CONDITIONAL GO` 已被真实资源库补测覆盖。详细证据见 `docs/U27_WINDOWS_GUI_ACCEPTANCE_RESULT.md`。

## 6. 当前主线：U28 资源库授权与真实 Index 闭环修复

**预计：1～2 轮。允许定向修改产品代码。**

### U28-A：根因定位与修复

必须统一以下状态来源：

```text
Windows 原生目录授权
→ Electron 安全 token / root snapshot
→ Settings 授权摘要
→ 读取已有 Index / 扫描能力
→ App 资源库状态
→ 首页、音声库、音乐库、PlayerBar
→ AI 维护真实诊断快照
```

修复要求：

1. 原生目录选择完成后，设置页必须立即识别已授权 root。
2. “读取已有记录”和“一键扫描并应用”必须根据真实授权能力启用。
3. 有 Index 时允许读取；无 Index 时允许安全扫描并生成。
4. 顶栏、设置页、浏览页和播放器不得使用互相独立的陈旧计数。
5. 诊断页必须读取真实授权/Index 状态；若该能力暂不可用，应禁用并明确说明，不能显示 Demo 为真实结果。
6. 不把真实绝对路径或 `file://` 暴露到 Renderer 日常界面。
7. 不修改真实媒体文件。

### U28-B：真实闭环复验

修复合入前必须完成：

```text
原生选择临时资源库
→ 读取已有 Index 或安全扫描
→ 音声库显示作品
→ 播放至少一个音轨
→ 关闭并重启
→ 状态一致
```

再使用真实 `E:\arsm` 做只读验证：

- 授权状态一致；
- 读取已有 Index 或明确可执行扫描；
- 浏览作品与顶栏计数一致；
- 播放一个音轨；
- 不执行清理、移动、覆盖或批量写入。

### U28 完成门槛

- MAJ-001 关闭。
- MAJ-002 关闭或降级为明确禁用、不会误导的非阻塞状态。
- 真实目录授权 → Index → 浏览 → 播放闭环通过。
- U02～U28 专项 verifier、完整稳定回归和生产构建通过。
- 用户配置恢复、Git clean、无残留进程。

在 U28 完成前，不进入字幕、导入器或发布候选验收。

## 10. 质量与安全硬规则

1. 所有任务从最新 `origin/main` 创建独立分支。
2. 不在 `main` 直接试验。
3. 不修改真实用户媒体做破坏性测试。
4. copy-only 优先；move-only 只能使用临时副本并二次确认。
5. 播放、导入、Index、备份、恢复和安装链必须保留稳定回归。
6. 日常 UI 不显示 MVP、verifier、IPC、Git、命令行或原始工程状态。
7. 真实诊断状态必须来自实际授权和 Index，不得使用 Demo 冒充。
8. 绝对路径和 `file://` 不进入 Renderer 日常界面。
9. 不为了代码整齐进行全项目重构。
10. verifier 优先验证行为和稳定合同，不绑定局部变量名。
11. high / critical 依赖风险阻止合并；moderate 显式记录。
12. 最终发布前必须通过 portable、NSIS、安装、卸载、进程回收和 SHA-256。

## 11. 路线变更程序

改变路线前必须暂停开发、审计当前 `main` 与开放 PR、说明价值和风险、先更新 `PROJECT_ROADMAP.md` 与 `PROJECT_STATE.md`，再创建执行分支。
