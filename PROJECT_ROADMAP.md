# PROJECT_ROADMAP

## 当前暂停点

```text
0.146.0-mvp108：导入器阶段收尾
```

暂停开发的目的：整理项目、做小样本回归、确认导入器是否足够稳定，再决定下一个大方向。

## 短期：只做检查，不开新功能

1. 小样本 copy-only 导入回归。
2. 小样本 move-only 导入回归。
3. 检查导入后首页 / 音声库 / 音乐库刷新。
4. 检查播放 / 字幕 / 外部打开未被破坏。
5. 检查主界面没有过多工程说明。
6. 检查没有 absolutePath / file:// 暴露到主界面。

## 下一阶段候选

| 方向 | 建议优先级 | 说明 |
|---|---:|---|
| 导入器小修 | 高 | 根据人工回归补 bug，不扩展大功能。 |
| 首页 / 播放器 UI 精修 | 中 | 像正常播放器产品，减少工程感。 |
| 元数据 Provider | 中 | 单个 RJ 查询→预览→手动确认→写入，不做全库自动抓。 |
| mpv 后端 | 中 | HTMLAudio 保底，mpv 子进程作为正式播放增强路线。 |
| SQLite | 后置 | 等导入/播放/元数据真实问题更清楚后再迁移。 |
| 下载器 | 后置 | 先做导入器和元数据，下载器最后做。 |

## 不建议立刻做

```text
不立刻开下载器
不立刻开 SQLite
不立刻开 mpv
不立刻做大批量 move
不立刻做自动元数据全库抓取
```

## Codex 策略

Codex 只在必要实机验证 / Git 合并 / 打包验收时使用。非必要不安排 Codex，以节省额度。
