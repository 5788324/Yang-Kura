# MVP-40 — 媒体库页面一致性收口

## 目标

MVP-40 的目标是让“音乐库”和“歌单页”更接近同一套媒体产品语言，而不是各自保留旧原型状态。

## 新增服务

```text
src/services/mediaSurfaceStatusService.ts
```

该服务集中生成：

- 音轨状态标签
- 专辑状态标签
- 歌单状态标签
- 音乐库统计摘要
- 歌单统计摘要

## 用户可见效果

### 音乐库

- 顶部新增专辑 / 音轨 / 本地资源 / 有字幕 / 已收藏统计。
- 歌曲列表显示“音乐 / 本地 / 示例 / 有字幕 / 外部打开”等状态。
- 专辑卡片显示统一状态标签。
- 文件夹视图改为媒体分组表述，不展示或暗示真实磁盘路径。

### 歌单页

- 顶部新增歌单 / 自建 / 音轨 / 音声 / 音乐统计。
- 歌单卡片显示“系统示例 / 本地歌单 / 音声 / 音乐 / 本地 / 空歌单”等状态。
- 详情页继续保持混合播放能力，但文案更偏用户使用场景。

## 安全边界

本轮只做 UI 和前端状态摘要，不新增真实文件能力。

禁止项仍然成立：

```text
不接 SQLite
不接下载器
不抓 ASMR.one / DLsite 元数据
不接 mpv
不删除文件
不移动文件
不重命名文件
不把 absolutePath / file:// 暴露给 renderer
```
