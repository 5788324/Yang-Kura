## 项目级 UI 硬规则（长期有效）

> 日常层只展示用户实际会使用的功能；诊断、回归、工程状态、测试入口、命令行说明、MVP/版本收口信息和检修工具统一进入 AI 维护或隐藏兼容层，不得长期污染主界面。

- 该规则不随 U24/U25 结束而失效，后续所有 UI 任务必须遵守；
- 有排障价值的底层能力可以保留，但默认不得成为日常卡片、窗口或一级入口；
- verifier 需要历史锚点时，必须使用 hidden 与 aria-hidden，不得重新显示给普通用户；
- 新增 UI 无法明确回答“用户日常是否会直接操作”时，默认进入 AI 维护；
- 详细规则见 docs/UI_DAILY_SURFACE_RULES.md。
